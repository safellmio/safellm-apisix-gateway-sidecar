# Contributing

Thanks for your interest in SafeLLM OSS.

## How to Contribute

1. Fork the repository and create a feature branch.
2. Make your changes with clear commit messages.
3. Run tests locally:
   - `./run_tests.sh`
4. Open a pull request with a concise description and rationale.

## Coding Standards

- Python 3.11+
- Prefer explicit, readable code over cleverness.
- Keep performance overhead minimal (security sidecar must stay fast).

## Reporting Issues

Please use GitHub Issues for bugs and feature requests.
For security issues, use `security@safellm.io`.
